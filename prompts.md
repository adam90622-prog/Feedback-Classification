# 피드백 분류 프롬프트

## System Prompt

```
당신은 카페 고객 피드백을 분석하는 전문가입니다.
지시한 JSON 형식으로만 응답하세요.
```

---

## User Prompt (피드백 업로드 시 동적 생성)

```
다음 고객 피드백 N건을 분석하세요.

[1] 음료가 전부 쏟아져서 왔어요 (별점: 1.0)
[2] 직원이 불친절했습니다
[3] 매장 분위기가 좋아요

분류 규칙:
1. 내용이 부정적이거나 문제를 호소하면 별점 유무와 관계없이 "불만"으로 분류.
   별점은 점수 보조용으로만 참고.
2. 불만이면 아래 리스크 유형 중 가장 심각한 하나를 선택.
3. 실제로 피해가 발생한 경우에만 2차피해유형에 포함
   (발생 가능성이 아닌 실제 발생).

유형: 불만 | 칭찬 | 문의 | 요청

리스크유형 (불만일 때만):
  - 법적·안전: 식품안전·알레르기·위생·신체 위험
  - 대량발생: 시스템 오류 등 다수 동시 피해 가능
  - 금전피해: 환불·결제·포인트 등 금전 손실
  - 평판·법적: 고소·SNS 확산·민원 등 외부 파급
  - 반복운영: 동일 문제 반복·운영 체계 부재
  - 서비스품질: 응대·대기·시설 등 품질 불편
  - 일반불편: 경미한 개인 불편

2차피해유형 (실제 피해 발생 시, 복수 가능):
  - 신체피해: 🚑 신체피해
  - 금전손실: 💸 금전손실
  - 법적조치: ⚖️ 법적조치

JSON 배열로만 응답 (다른 설명 없이):
[
  {"id": 1, "유형": "불만", "리스크유형": "법적·안전", "2차피해유형": ["신체피해"], "긴급이유": "알레르기 실제 발생, 법적 책임 위험"},
  {"id": 2, "유형": "칭찬", "리스크유형": "", "2차피해유형": [], "긴급이유": ""}
]
```

---

## 스코어링 기준

긴급도 = 리스크 기본점수 + 2차 피해 추가점수 + 별점 보정

### 리스크 유형별 기본 점수
| 리스크 유형 | 점수 |
|---|---|
| 법적·안전 | 10 |
| 대량발생 | 8 |
| 금전피해 | 7 |
| 평판·법적 | 6 |
| 반복운영 | 5 |
| 서비스품질 | 3 |
| 일반불편 | 1 |

### 2차 피해 추가 점수
| 피해 유형 | 추가점수 |
|---|---|
| 신체피해 (실제 발생) | +5 |
| 금전손실 (실제 발생) | +3 |
| 법적조치 의향 | +2 |

### 별점 보정 (공개 노출 리스크)
| 별점 | 추가점수 |
|---|---|
| ★1 | +3 |
| ★2 | +2 |
| ★3 | +1 |
| 없음 | +0 |
