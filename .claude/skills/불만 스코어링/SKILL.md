# 불만 스코어링 스킬

## Description

고객 피드백 CSV를 업로드받아 자동 분류하고, 리스크 기반 긴급도 점수를 산출하는 Streamlit 대시보드를 처음부터 만든다.

**트리거 키워드 (이렇게 말하면 이 스킬이 작동):**
- `/불만분류 [서비스명]`
- "불만 분류 만들어줘"
- "피드백 스코어링 대시보드"
- "고객 불만 대시보드 만들어줘"
- "피드백 분류 시스템 만들어줘"

---

## 사용법

```
/불만분류 [서비스 종류]
```

| 예시 입력 | 결과 |
|---|---|
| `/불만분류 카페` | 카페 고객 피드백 분류 대시보드 생성 |
| `/불만분류 배달앱` | 배달앱 특화 리스크 유형으로 조정 후 생성 |
| "피드백 분류 스킬로 병원 리뷰 시스템 만들어줘" | 병원 도메인으로 키워드·리스크 유형 조정 후 생성 |

---

## 절차 체크리스트

### 1단계 — 도메인 파악
- [ ] 서비스 종류 확인 (카페 / 배달 / 병원 / 커머스 등)
- [ ] 주요 불만 유형 예상 목록 사용자와 합의
- [ ] CSV 컬럼 구조 확인 (내용 필수 / 별점·날짜·경로 선택)

### 2단계 — 분류 체계 설계
- [ ] 유형 4가지 정의: 불만 / 칭찬 / 문의 / 요청
- [ ] 리스크 유형 7단계 정의 (도메인에 맞게 키워드 조정)
  - 법적·안전 (10점) → 대량발생 (8) → 금전피해 (7) → 평판·법적 (6) → 반복운영 (5) → 서비스품질 (3) → 일반불편 (1)
- [ ] 2차 피해 유형 정의: 신체피해 (+5) / 금전손실 (+3) / 법적조치 (+2)
- [ ] 별점 보정 적용 여부 결정 (★1→+3, ★2→+2, ★3→+1)

### 3단계 — classifier.py 작성
- [ ] `RISK_CONFIG` / `SECONDARY_CONFIG` 딕셔너리 정의
- [ ] `_compute_score(risk_type, secondary, star)` 작성
- [ ] LLM 프롬프트 작성 (System + User, JSON 배열 응답 형식)
  - 분류 규칙: 별점 무관 내용 우선, 2차 피해는 실제 발생만
  - Few-shot: 불만 1건 + 칭찬 1건 예시 포함
- [ ] 룰기반 폴백 작성 (정규식 키워드 → 도메인 맞게 수정)
  - `RISK_PATTERNS` / `SECONDARY_PATTERNS` / `NEGATIVE_PATTERNS`
- [ ] `classify()` 진입점: LLM 시도 → 실패 시 룰기반 폴백

### 4단계 — streamlit_app.py 작성
- [ ] CSV 업로드 + 컬럼 자동 감지 (별칭 목록으로 유연하게)
- [ ] 날짜 정규화: `MM-DD` 또는 `MM-DD-YYYY` 형식 통일
- [ ] Top3 급한 불만 카드 (리스크 태그 + 2차피해 뱃지 + 점수 표시)
- [ ] 유형별 통계 차트 (Plotly 막대)
- [ ] 전체 피드백 테이블 (필터 + 정렬)
- [ ] 완료 처리 UI
  - 버튼 클릭 → `pending_complete` session_state 저장 → 확인창 표시
  - TF-IDF 유사도(char_wb, ngram 2~3)로 유사 불만 탐색
  - 채널별 그룹핑 + 체크박스 개별 선택
  - "선택한 것만 완료" / "모두 완료" / "이 항목만" / "취소"
  - 완료 후 `st.toast()`로 알림

### 5단계 — 배포
- [ ] `requirements.txt` 작성 (streamlit / pandas / plotly / requests / scikit-learn)
- [ ] `.gitignore` 작성 (.env / *.csv / __pycache__ / .streamlit/secrets.toml)
- [ ] GitHub 레포 생성 후 push
- [ ] Streamlit Community Cloud 배포
  - share.streamlit.io → GitHub 연결 → secrets에 `ANTHROPIC_API_KEY` 등록

## 입출력 예시

### 예시 1 — 카페

**입력:**
```
/불만분류 카페
```

**Claude 진행 순서:**
1. 카페 도메인 리스크 키워드 확인 (알레르기, 이물질, 결제 오류, 진동벨 등)
2. `classifier.py` + `streamlit_app.py` 생성
3. GitHub push + Streamlit Cloud 배포 안내

**출력 결과물:**
- `classifier.py` — LLM 프롬프트 + 룰기반 분류 엔진
- `streamlit_app.py` — 대시보드 앱
- `requirements.txt` / `.gitignore`
- 배포 URL (Streamlit Cloud)

---

### 예시 2 — 배달앱 (도메인 조정)

**입력:**
```
피드백 분류 스킬로 배달앱 고객 불만 대시보드 만들어줘
```

**Claude 조정 내용:**
- 리스크 키워드: 카페(음료·매장) → 배달앱(배송지연·음식누락·포장파손)으로 교체
- 2차 피해 패턴: 식중독·부상 외에 배송사고 추가
- 채널 예시: 앱리뷰 / 고객센터 / SNS

## 핵심 설계 원칙

1. **LLM은 분류만, 점수는 코드에서** — 점수 공식을 코드에 두면 조정이 쉽다
2. **룰기반은 폴백** — API 없어도 최소 동작 보장
3. **완료 처리는 확인 후** — 자동 일괄처리는 오류 유발, 사용자가 보고 선택
4. **별점은 노출 리스크 보정** — 같은 불만이라도 공개 채널 우선 대응
